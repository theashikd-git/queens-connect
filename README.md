# 🏥 Queens Connect — Hospital Field Marketing App

A production-ready Flutter + Firebase application for tracking hospital field marketing visits with GPS verification.

## Key Features

### For Field Users
- **Auto GPS capture** on form open (cannot be edited)
- **Mock GPS detection** — flags suspicious location spoofing
- **Accuracy validation** — blocks submission if GPS accuracy > 50m
- **Hospital selection** from Firestore-backed dropdown
- **Photo evidence** via camera or gallery
- **Clean visit form** with purpose categories

### For Managers
- **Real-time visit stream** with status filtering
- **Stats dashboard** — total, valid, warning, suspicious counts
- **Compliance rate** visualization
- **Visit detail view** with:
  - Side-by-side manual vs GPS comparison
  - OpenStreetMap with two markers (claimed vs actual)
  - Dotted line showing the gap
  - Photo viewer
  - Mock GPS flag alerts

### Location Verification
| Distance | Status | Meaning |
|----------|--------|---------|
| < 100m | ✅ Valid | At the hospital |
| 100–300m | ⚠️ Warning | Nearby but not on-site |
| > 300m | 🚨 Suspicious | Not at the claimed location |

## Quick Start

```bash
# 1. Clone and install
flutter pub get

# 2. Set up Firebase (see FIREBASE_SETUP.md)
flutterfire configure --project=your-project-id

# 3. Run
flutter run
```

See **FIREBASE_SETUP.md** for complete setup including security rules, indexes, and seeding.
